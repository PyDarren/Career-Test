## 1. 测评模型与三层标签定义

### 1.1 模型总体架构

采用 **IPIP 大五人格 (OCEAN) + 霍兰德 RIASEC** 双框架一体化测评方案。全部 80 题在同一答题流程中完成，大五人格 50 题测量人格特质，RIASEC 30 题测量职业兴趣，两套量表独立计分后融合生成三层人格标签。

```

│  │  如 IAS      │  │  VSN-FHR     │  │  ●●●●●       │    │
```

| 量表来源 | 自编（无科学验证） | IPIP-50（公有领域，有信效度数据） |

### 1.2 大五人格 (IPIP-50) — 五维度特质测量

基于 IPIP（International Personality Item Pool）大五人格量表，50 题覆盖五个维度，每维度 10 题，部分反向计分。

| 维度 | 英文 | dimension_prefix | 高分特征 | 低分特征 | 题目数 |
|------|------|-----------------|---------|---------|--------|
| 开放性 | Openness (O) | BO | 好奇、创造、求新 | 务实、传统、保守 | 10 |
| 尽责性 | Conscientiousness (C) | BC | 自律、计划、负责 | 灵活、随性、自发 | 10 |
| 外向性 | Extraversion (E) | BE | 热情、社交、活跃 | 内敛、安静、独处 | 10 |
| 宜人性 | Agreeableness (A) | BA | 亲和、合作、信任 | 独立、理性、竞争 | 10 |
| 神经质 | Neuroticism (N) | BN | 敏感、焦虑、情绪化 | 稳定、冷静、自信 | 10 |

**计分方式**：7 点李克特量表，每维度 10 题之和（范围 10-70），部分题目反向计分（翻转规则：1↔7, 2↔6, 3↔5, 4↔4）。

**商业价值**：大五人格五维度的高/中/低组合生成 243 种人格原型（3^5），驱动人格画像名与色彩光谱码，是三层标签中第二层和第三层的数据来源。

**题目设计示例**：

| 题号 | dimension_prefix | 陈述句 | 计分方向 |
|------|-----------------|--------|---------|
| Q001 | BO | 你喜欢思考新的想法和可能性 | 正向 |
| Q003 | BO | 你更倾向于按部就班而非尝试新方法 | 反向 |
| Q011 | BC | 你会提前制定计划并按计划执行 | 正向 |
| Q021 | BE | 在社交场合中你通常是主动交谈的人 | 正向 |
| Q031 | BA | 即使意见不同，你也能理解对方的立场 | 正向 |
| Q041 | BN | 你经常感到焦虑或不安 | 正向 |

### 1.3 职业兴趣 (RIASEC-30) — 六类型兴趣测量

基于霍兰德 (John L. Holland) 职业兴趣理论，30 题覆盖六个类型，每类型 5 题，全部正向计分。题项均为自主编写，不使用 SDS 官方量表。

| 类型 | 英文 | dimension_prefix | 核心特征 | 典型职业 | 题目数 |
|------|------|-----------------|---------|---------|--------|
| 现实型 | Realistic (R) | RR | 喜欢动手操作、使用工具 | 工程师、技术员、运动员 | 5 |
| 研究型 | Investigative (I) | RI | 喜欢分析、研究、解决复杂问题 | 科学家、分析师、医生 | 5 |
| 艺术型 | Artistic (A) | RA | 喜欢创造、表达、审美设计 | 设计师、作家、音乐家 | 5 |
| 社会型 | Social (S) | RS | 喜欢帮助、教导、服务他人 | 教师、咨询师、社工 | 5 |
| 企业型 | Enterprising (E) | RE | 喜欢领导、说服、影响他人 | 管理者、销售、律师 | 5 |
| 常规型 | Conventional (C) | RC | 喜欢组织、整理、处理数据 | 会计、行政、程序员 | 5 |

**计分方式**：7 点李克特量表，每类型 5 题之和（范围 5-35），全部正向计分，无反向题。

**商业价值**：RIASEC 六类型得分降序排列取前三，生成 3 字母职业兴趣码（如 `IAS`），120 种组合。RIASEC 码本身即职业兴趣的直接表达，与大五原型结合可生成 29160 种独特人格标签，是三层标签中第一层的数据来源。

**题目设计示例**：

| 题号 | dimension_prefix | 陈述句 | 计分方向 |
|------|-----------------|--------|---------|
| Q051 | RR | 你喜欢动手修理或组装物品 | 正向 |
| Q056 | RI | 你喜欢分析复杂数据寻找规律 | 正向 |
| Q061 | RA | 你喜欢通过创意作品表达自我 | 正向 |
| Q066 | RS | 帮助他人成长让你感到充实 | 正向 |
| Q071 | RE | 你喜欢带领团队达成目标 | 正向 |
| Q076 | RC | 你喜欢整理信息让一切井井有条 | 正向 |

### 1.4 三层人格标签体系

用户完成 80 题测评后，系统生成三层标签，共同构成用户的人格身份标识：

| 层级 | 标签类型 | 生成逻辑 | 示例 | 数量 | 可见性 |
|------|---------|---------|------|------|--------|
| 第一层 | RIASEC 职业兴趣码 | 六维度得分降序取前三，并列按 R>I>A>S>E>C 优先 | `IAS` | 120 种组合 | 免费可见 |
| 第二层 | 人格画像名 | 大五五维度高/中/低三分组合映射 243 种原型 | `VSN-FHR` | 243 种原型 | 免费可见 |
| 第三层 | 色彩光谱码 | 五维度百分位映射为 5 色圆点序列 | ●●●●● | 连续唯一 | 免费可见 |

#### 1.4.1 色彩光谱设计

每个维度分配专属色，颜色深浅代表得分高低（百分位映射为 5 档色深）：

| 维度 | 专属色系 | 基准色值 | 高分色（深） | 低分色（浅） |
|------|---------|---------|-------------|-------------|
| 开放性 (O) | 紫色系 | `#9B7ED8` | `#6B4EAB` | `#D4C3F0` |
| 尽责性 (C) | 蓝色系 | `#5a96b1` | `#3A7691` | `#B0D4E0` |
| 外向性 (E) | 绿色系 | `#5ea67e` | `#3E865E` | `#B0D5C0` |
| 宜人性 (A) | 金色系 | `#deb45c` | `#BE943C` | `#F0D9A0` |
| 稳定性 (N↓) | 珊瑚色系 | `#e17055` | `#C15035` | `#F0A590` |

**5 档色深映射规则**（基于百分位）：

| 色深档位 | 百分位范围 | 含义 | O 色值 | C 色值 | E 色值 | A 色值 | N 色值 |
|---------|----------|------|--------|--------|--------|--------|--------|
| Level 1 | 0-20% | 极低 | `#D4C3F0` | `#B0D4E0` | `#B0D5C0` | `#F0D9A0` | `#F0A590` |
| Level 2 | 20-40% | 偏低 | `#B89FE0` | `#8AB5C6` | `#88BD9F` | `#E5C67C` | `#E88870` |
| Level 3 | 40-60% | 中等 | `#9B7ED8` | `#5a96b1` | `#5ea67e` | `#deb45c` | `#e17055` |
| Level 4 | 60-80% | 偏高 | `#8266C2` | `#4A86A1` | `#4E966E` | `#CEA44C` | `#D16045` |
| Level 5 | 80-100% | 极高 | `#6B4EAB` | `#3A7691` | `#3E865E` | `#BE943C` | `#C15035` |

> 神经质 (N) 维度：高分=情绪敏感（深色），低分=情绪稳定（浅色）。色彩深浅直观反映维度强度，不带有价值判断。

### 1.6 人格认证卡设计规范（基于 `simple.png` 参考稿）

## 3. 计分引擎与三层标签生成算法 (Scoring Engine)

### 3.1 7 点李克特量表计分

所有题目共用 7 点李克特量表，正向题直接取量表值（1-7），反向题进行翻转：

```python
# scoring/calculators.py

# 反向计分翻转映射
REVERSE_MAPPING: dict[int, int] = {
    1: 7,   # 完全不符合 ↔ 非常符合
    2: 6,   # 比较不符合 ↔ 比较符合
    3: 5,   # 有些不符合 ↔ 有些符合
    4: 4,   # 不确定 ↔ 不确定
    5: 3,   # 有些符合 ↔ 有些不符合
    6: 2,   # 比较符合 ↔ 比较不符合
    7: 1,   # 非常符合 ↔ 完全不符合
}


def get_effective_score(
    scale_value: int,
    is_reverse: bool,
) -> int:
    """
    获取有效计分值。

    正向题：直接返回 scale_value (1-7)
    反向题：翻转后返回 (1↔7, 2↔6, 3↔5, 4↔4)
    """
    if is_reverse:
        return REVERSE_MAPPING[scale_value]
    return scale_value
```

### 3.2 OCEAN 原始分计算

```python
def calculate_ocean_raw_score(
    answers: list[UserAnswerLog],
    config: DimensionConfig,
) -> int:
    """
    大五人格单维度原始分计算。

    每维度 10 题，满分 70 分，最低 10 分。
    raw_score = sum(effective_score) ∈ [10, 70]

    支持反向题：reverse_question_ids 中的题目翻转计分。
    """
    reverse_ids = set(config.reverse_question_ids)
    total = 0

    for q_id in config.question_ids:
        answer = _find_answer(answers, q_id)
        is_reverse = q_id in reverse_ids
        effective = get_effective_score(answer.scale_value, is_reverse)
        total += effective

    return total  # 范围 10-70
```

**数学公式**（每维度 $N=10$ 题）：

$$\text{raw\_score}_D = \sum_{i=1}^{N} s_i \times f(r_i), \quad s_i \in \{1,2,3,4,5,6,7\}$$

其中 $f(r_i)$ 为反向翻转函数：$r_i = \text{True}$ 时 $f = (8 - s_i)$，$r_i = \text{False}$ 时 $f = s_i$。

取值范围 $[N, 7N] = [10, 70]$。

### 3.3 RIASEC 原始分计算

```python
def calculate_riasec_raw_score(
    answers: list[UserAnswerLog],
    config: RIASECTypeConfig,
) -> int:
    """
    RIASEC 单类型原始分计算。

    每类型 5 题，满分 35 分，最低 5 分。
    全部正向计分，无反向题。
    raw_score = sum(scale_value) ∈ [5, 35]
    """
    total = 0
    for q_id in config.question_ids:
        answer = _find_answer(answers, q_id)
        total += answer.scale_value  # RIASEC 全部正向，无需翻转

    return total  # 范围 5-35
```

### 3.4 标准化转换

#### 百分位转换

```python
from scipy.stats import norm

def raw_to_percentile(raw_score: int, norm_mean: float, norm_std: float) -> float:
    """
    将原始分转换为百分位（基于中国用户常模）。

    百分位含义：在该维度上，用户得分高于常模群体中百分之多少的人。
    """
    z = (raw_score - norm_mean) / norm_std
    return round(norm.cdf(z) * 100, 1)
```

#### T 分数转换

$$T = 50 + 10 \times \frac{\text{raw\_score} - \mu_D}{\sigma_D}$$

$\mu_D$、$\sigma_D$ 从配置文件加载（基于中国用户常模）。

#### 高中低三分划分

```python
def determine_level_label(
    percentile: float,
    low_threshold: float = 33.3,
    high_threshold: float = 66.7,

    """
    三级分制划分高(H)/中(M)/低(L)。
    percentile < 33.3 → "L" (低)
    33.3 <= percentile <= 66.7 → "M" (中)
    percentile > 66.7 → "H" (高)

    if percentile < low_threshold:
        return "L"
    elif percentile > high_threshold:
```

### 3.5 243 原型匹配逻辑（核心）

```python
# scoring/archetype_matcher.py

def match_archetype(
    ocean_scores: list[OCEANDimensionScore],
    archetype_configs: dict,
) -> dict:
    """
    243 种原型匹配。

    逻辑：
    1. 五维度各自按三分位划分高(H)/中(M)/低(L)
    2. 生成 5 字符组合码（如 "HHMHL"）
    3. 从 243 种原型配置中查找匹配的原型

    3^5 = 243 种唯一组合，每种组合对应一个原型。
    """
    # 1. 生成维度组合码
    code_parts: list[str] = []
    for dim_score in ocean_scores:
        code_parts.append(dim_score.level_label)
    archetype_code = "".join(code_parts)  # 如 "HHMHL"

    # 2. 查找原型配置
    for archetype_id, config in archetype_configs.items():
        if config["archetype_code"] == archetype_code:
            return config

    raise ValueError(f"No archetype found for code: {archetype_code}")
```

**原型匹配示意**：

```
大五五维度 → 三分位划分 → 组合码 → 原型

O: 85% → H ─┐
C: 72% → H ─┤
E: 50% → M ─┼→ "HHMHL" → 原型: VSN-SRN
A: 58% → H ─┤
N: 20% → L ─┘
```

### 3.6 RIASEC 兴趣码生成

```python
# scoring/engine.py

# RIASEC 并列时的优先级顺序
RIASEC_PRIORITY = ["R", "I", "A", "S", "E", "C"]

def generate_riasec_code(
    riasec_scores: list[RIASECDimensionScore],
) -> str:
    """
    生成 3 字母 RIASEC 职业兴趣码。

    逻辑：
    1. 六类型按原始分降序排列
    2. 取前三名
    3. 前三名中如有并列分，按 R>I>A>S>E>C 优先级排序

    120 种唯一组合 (P(6,3) = 6×5×4 = 120)。
    """
    # 1. 按原始分降序排列，并列时按 RIASEC 优先级排序
    sorted_scores = sorted(
        riasec_scores,
        key=lambda s: (-s.raw_score, RIASEC_PRIORITY.index(s.type.value)),
    )

    # 2. 取前三名
    top_three = sorted_scores[:3]

    # 3. 生成 3 字母码
    riasec_code = "".join(s.type.value for s in top_three)

    return riasec_code  # 如 "IAS"
```

**RIASEC 并列处理示例**：

```
得分: R=20, I=20, A=18, S=15, E=12, C=10
→ R 和 I 并列第一 (20分)，按优先级 R > I
→ 第三名为 A (18分)
→ 兴趣码: "RIA"

得分: R=15, I=18, A=18, S=18, E=12, C=10
→ I, A, S 并列第二 (18分)，按优先级 I > A > S
→ 取 I, A 作为第二、三名
→ 兴趣码: "RIA"
```

```python
COLOR_TABLE: dict[str, list[str]] = {

    "C": ["#B0D4E0", "#8AB5C6", "#5a96b1", "#4A86A1", "#3A7691"],

    "N": ["#F0A590", "#E88870", "#e17055", "#D16045", "#C15035"],
}

def generate_color_spectrum(
    ocean_scores: list[OCEANDimensionScore],
    color_levels: list[dict],
) -> ColorSpectrum:
    """
    生成色彩光谱码。

    逻辑：
    1. 每维度百分位 → 色深档位 (1-5)
    2. 档位 → 对应 HEX 色值
    3. 生成 5 色圆点序列

    输出：5 个 SpectrumDot + 可视化字符串
    """
    dots: list[SpectrumDot] = []

    for dim_score in ocean_scores:
        level = percentile_to_level(dim_score.percentile, color_levels)
        color = COLOR_TABLE[dim_score.dimension.value][level - 1]

        dots.append(SpectrumDot(
            dimension=dim_score.dimension,
            percentile=dim_score.percentile,
            level=level,
            color=color,
        ))

    # 生成可视化字符串（5个圆点，大小代表色深档位）
    visual = "".join(
        "●" if d.level >= 3 else "◦" for d in dots
    )

    return ColorSpectrum(dots=dots, visual=visual)
```

### 3.8 职业推荐匹配算法

职业匹配综合考虑**人格原型 + RIASEC 兴趣码**双维度：

```python
# scoring/career_matcher.py

def match_careers(
    archetype_id: int,
    archetype_name: str,
    riasec_code: str,
    ocean_scores: list[OCEANDimensionScore],
    career_config: dict,
) -> list[CareerMatch]:
    """
    原型 + RIASEC 双驱动职业推荐。

    匹配公式:
    match_score = archetype_affinity (0-50)
               + riasec_match (0-35)
               + clarity_bonus (0-15)

    archetype_affinity: 原型与职业的基础匹配度
    riasec_match: 用户 RIASEC 码与职业 RIASEC 码的重合度
    clarity_bonus: 维度偏好清晰度加成
    """
    matches: list[CareerMatch] = []

    # 计算维度清晰度（偏离中位数的程度）
    avg_clarity = sum(abs(s.percentile - 50) for s in ocean_scores) / len(ocean_scores)
    clarity_bonus = min(15, avg_clarity / 3.3)

    for career_id, career_data in career_config.items():
        # 1. 原型亲和度 (0-50)
        archetype_affinity = career_data["archetype_affinity"].get(archetype_id, 20)

        # 2. RIASEC 匹配度 (0-35)
        career_riasec = career_data["riasec_code"]  # 如 "IAS"
        riasec_match = _calc_riasec_match(riasec_code, career_riasec)

        # 3. 总分
        total = archetype_affinity + riasec_match + clarity_bonus
        total = min(100, total)

        match_level = "high" if total >= 75 else "medium" if total >= 50 else "low"

        matches.append(CareerMatch(
            career_id=career_id,
            career_name=career_data["name"],
            match_score=round(total, 1),
            match_level=match_level,
            tags=career_data.get("tags", []),
            recommended_archetype=archetype_name,
            recommended_riasec=riasec_code,
        ))

    matches.sort(key=lambda m: m.match_score, reverse=True)
    return matches[:10]  # 返回 Top 10


def _calc_riasec_match(user_code: str, career_code: str) -> int:
    """
    RIASEC 码匹配度计算 (0-35)。

    匹配规则：
    - 3 字母完全相同且顺序一致 → 35
    - 3 字母相同但顺序不同 → 28
    - 前 2 字母相同 → 22
    - 前 1 字母相同 → 15
    - 无匹配 → 5
    """
    user_set = set(user_code)
    career_set = set(career_code)

    if user_code == career_code:
        return 35
    elif user_set == career_set:
        return 28
    elif user_code[:2] == career_code[:2]:
        return 22
    elif user_code[0] == career_code[0]:
        return 15
    elif len(user_set & career_set) >= 2:
        return 12
    elif len(user_set & career_set) >= 1:
        return 8
    else:
        return 5
```

```python
# scoring/validity.py

def check_lie_scale(
    answers: list[UserAnswerLog],
    lie_scale_ids: list[str],
    threshold: int = 6,
) -> bool:
    """
    测谎题检测。

    逻辑：社会赞许性题目选 6-7（比较符合/非常符合）→ +1 谎言分。
    判定阈值：谎言分 >= 2 → lie_scale_triggered = True
    """
    lie_score = 0

    for q_id in lie_scale_ids:
        answer = _find_answer(answers, q_id)
        if answer.scale_value >= threshold:
            lie_score += 1

    return lie_score >= 2
```

**判定阈值**：谎言分 >= 2 → `lie_scale_triggered = True`

### 4.2 矛盾题对检测

3 对一致性检测题，每对包含一道正向题和一道反向题，测量同一维度：

| 配对 | 正向题 | 反向题 | 维度 | 矛盾判定 |
|------|--------|--------|------|---------|
| 配对1 | Q021 "在社交场合中你通常是主动交谈的人" | Q025 "你更喜欢独处而非与人交往" | E | 两题均选 4-5 或均选 1-2 → 矛盾 |
| 配对2 | Q031 "即使意见不同，你也能理解对方的立场" | Q035 "你经常与他人产生争执" | A | 两题均选 4-5 或均选 1-2 → 矛盾 |
| 配对3 | Q041 "你经常感到焦虑或不安" | Q045 "你很少感到紧张或担忧" | N | 两题均选 4-5 或均选 1-2 → 矛盾 |

```python
def check_consistency(
    answers: list[UserAnswerLog],
    consistency_pairs: list[ConsistencyPair],
) -> bool:
    """
    矛盾题对一致性检测。

    逻辑：正向题和反向题应处于量表对称位置。
    - 正向题选 4-5（同意正向陈述）↔ 反向题应选 1-2（不同意反向陈述）
    - 若两题同侧（均选 4-5 或均选 1-2），计为矛盾。

    判定：3 对中 >= 2 对矛盾 → consistency_failed = True
    """
    conflict_count = 0

    for pair in consistency_pairs:
        ans_a = _find_answer(answers, pair.item_a)  # 正向题
        ans_b = _find_answer(answers, pair.item_b)  # 反向题

        a_high = ans_a.scale_value >= 4   # 同意正向
        a_low = ans_a.scale_value <= 2    # 不同意正向
        b_high = ans_b.scale_value >= 4   # 同意反向
        b_low = ans_b.scale_value <= 2    # 不同意反向

        # 两题同侧 = 矛盾（都同意或都不同意）
        if (a_high and b_high) or (a_low and b_low):
            conflict_count += 1

    return conflict_count >= 2
```

### 4.3 行为异常检测

基于 `UserAnswerLog` 中的 `response_duration_ms` 和 `modification_count`：

```python
def detect_behavior_anomaly(answers: list[UserAnswerLog]) -> BehaviorAnomaly:
    durations = [a.response_duration_ms for a in answers]
    values = [a.scale_value for a in answers]

    # 1. 响应过快：平均耗时 < 1500ms
    avg_duration = sum(durations) / len(durations)
    too_fast = avg_duration < 1500

    # 2. 直线作答：同一选项占比 > 80%
    from collections import Counter
    value_counts = Counter(values)
    max_ratio = max(value_counts.values()) / len(values)
    straight_lining = max_ratio > 0.80

    # 3. 交替模式检测：如 1,5,1,5,... 连续 10+ 题
    alternating = _detect_alternating_pattern(values)

    # 4. 响应时间方差过低：std < 500ms（自动化工具特征）
    import statistics
    duration_std = statistics.stdev(durations) if len(durations) > 1 else 0
    too_uniform = duration_std < 500

    return BehaviorAnomaly(
        too_fast=too_fast,
        straight_lining=straight_lining,
        alternating=alternating,
        too_uniform=too_uniform,
    )
```

**异常判定阈值表（配置化）**：

| 检测项 | 阈值 | 触发动作 |
|--------|------|---------|
| 单题平均耗时 | < 1500 ms | `response_time_anomaly = True` |
| 同一选项占比 | > 80% | `straight_lining_detected = True` |
| 交替模式 | 连续 10+ 题完美交替 | `response_time_anomaly = True` |
| 响应时间标准差 | < 500 ms | `response_time_anomaly = True` |
| 测谎题得分 | >= 2/3 | `lie_scale_triggered = True` |
| 矛盾题对冲突 | >= 2/3 | `consistency_failed = True` |

### 4.4 效度标记与置信度计算

```python
def compute_validity(
    answers: list[UserAnswerLog],
    config: ValidityConfig,
) -> ValidityResult:
    lie_triggered = check_lie_scale(answers, config.lie_scale_ids, config.lie_scale_threshold)
    consistency_failed = check_consistency(answers, config.consistency_pairs)
    anomaly = detect_behavior_anomaly(answers)

    invalid_reasons: list[str] = []
    penalty = 0.0

    if lie_triggered:
        invalid_reasons.append("lie_scale_triggered")
        penalty += 0.3
    if consistency_failed:
        invalid_reasons.append("consistency_failed")
        penalty += 0.25
    if anomaly.too_fast or anomaly.too_uniform or anomaly.alternating:
        invalid_reasons.append("response_time_anomaly")
        penalty += 0.2
    if anomaly.straight_lining:
        invalid_reasons.append("straight_lining_detected")
        penalty += 0.15

    is_valid = len(invalid_reasons) == 0
    confidence = max(0.0, 1.0 - penalty)

    return ValidityResult(
        is_valid=is_valid,
        invalid_reasons=invalid_reasons,
        confidence_score=confidence,
        lie_scale_triggered=lie_triggered,
        response_time_anomaly=anomaly.too_fast or anomaly.too_uniform,
        straight_lining_detected=anomaly.straight_lining,
        consistency_failed=consistency_failed,
    )
```

**效度结果处理策略**：

| `is_valid` | `confidence_score` | 处理策略 |
|-----------|-------------------|---------|
| True | >= 0.8 | 正常出报告 |
| True | 0.5 - 0.8 | 出报告但标注"结果仅供参考" |
| False | < 0.5 | 提示"测评效度不足，建议重新测试"，不出报告 |

```python
# scoring/engine.py

class ScoringEngineInput(BaseModel):
    """计分引擎输入（纯数据，无框架依赖）"""
    answers: list[UserAnswerLog]         # 80 条答题记录
    question_bank_config: dict           # 题库配置（OCEAN + RIASEC）
    scoring_config: dict                 # 计分配置（量表、常模、色深档位）
    archetype_configs: dict              # 243 种原型配置
    validity_config: dict                # 效度检测配置
    career_config: dict                  # 职业推荐配置

class ScoringEngineOutput(BaseModel):
    """计分引擎输出（纯数据，无框架依赖）"""
    # 原型信息（免费可见）
    archetype_id: int                    # 原型编号 (1-243)
    archetype_name: str                  # 画像名，如 "VSN-FHR"
    archetype_slogan: str                # 一句话描述
    archetype_code: str                  # 维度组合码，如 "HHHHL"
    # 三层标签（免费可见）
    riasec_code: str                     # RIASEC码，如 "IAS"
    color_spectrum: ColorSpectrum        # 色彩光谱码
    # OCEAN 得分（免费可见）
    ocean_scores: list[OCEANDimensionScore]
    # RIASEC 得分（免费可见）
    riasec_scores: list[RIASECDimensionScore]
    # 免费卡片数据（免费可见，参照 simple.png）
    free_card_data: dict
    # 效度与置信度
    validity: ValidityResult
    confidence: float
    # 付费层内容
    career_matches: list[CareerMatch]    # 付费
    deep_report_outline: dict            # 付费
    computed_at: str


def calculate_assessment_result(
    input_data: ScoringEngineInput,
) -> ScoringEngineOutput:
    """
    测评计分引擎唯一入口函数。

    特性：
    - 纯函数：相同输入 → 相同输出（计分确定性）
    - 无副作用：不访问数据库、不调用网络、不写文件
    - 无框架依赖：不导入 Django / FastAPI 任何模块
    - 全配置驱动：所有规则从 input_data 中的 config 读取

    流程：
    1. 输入校验（80 题完整性）
    2. 效度检测（测谎题 + 矛盾题 + 行为异常）
    3. OCEAN 计分（5 维度 × 10 题 → 原始分 → 百分位 → 高/中/低三分划分）
    4. RIASEC 计分（6 类型 × 5 题 → 原始分 → 降序 → 取前三）
    5. 243 原型匹配（五维度高/中/低组合 → 原型名）
    6. 色彩光谱码生成（百分位 → 5色 + 色深）
    7. 职业推荐匹配（原型 + RIASEC 码双驱动）
    8. 报告大纲生成
    """
    # 1. 输入校验
    _validate_answers(input_data.answers, input_data.question_bank_config)

    # 2. 效度检测
    validity = compute_validity(input_data.answers, input_data.validity_config)
    confidence = validity.confidence_score

    # 3. OCEAN 计分
    ocean_scores: list[OCEANDimensionScore] = []
    for dim in OCEANDimension:
        dim_config = input_data.scoring_config["ocean_dimensions"][dim.value]
        raw_score = calculate_ocean_raw_score(input_data.answers, dim_config)
        percentile = raw_to_percentile(raw_score, dim_config["norm_mean"], dim_config["norm_std"])
        level_label = determine_level_label(
            percentile,
            input_data.scoring_config["scoring"]["low_threshold"],
            input_data.scoring_config["scoring"]["high_threshold"],
        )
        level = percentile_to_level(percentile, input_data.scoring_config["scoring"]["color_levels"])
        t_score = 50 + 10 * (raw_score - dim_config["norm_mean"]) / dim_config["norm_std"]

        ocean_scores.append(OCEANDimensionScore(
            dimension=dim,
            raw_score=raw_score,
            percentile=percentile,
            level_label=level_label,
            level=level,
            t_score=round(t_score, 1),
        ))

    # 4. RIASEC 计分
    riasec_scores: list[RIASECDimensionScore] = []
    for rtype in RIASECType:
        type_config = input_data.scoring_config["riasec_types"][rtype.value]
        raw_score = calculate_riasec_raw_score(input_data.answers, type_config)
        riasec_scores.append(RIASECDimensionScore(
            type=rtype,
            raw_score=raw_score,
            rank=0,       # 待排序后填入
            is_top_three=False,
        ))

    # 4.1 RIASEC 排序与排名
    sorted_riasec = sorted(
        riasec_scores,
        key=lambda s: (-s.raw_score, RIASEC_PRIORITY.index(s.type.value)),
    )
    for i, score in enumerate(sorted_riasec):
        score.rank = i + 1
        score.is_top_three = i < 3
    riasec_code = generate_riasec_code(riasec_scores)

    # 5. 243 原型匹配
    archetype_config = match_archetype(ocean_scores, input_data.archetype_configs)

    # 6. 色彩光谱码生成
    color_spectrum = generate_color_spectrum(
        ocean_scores,
        input_data.scoring_config["scoring"]["color_levels"],
    )

    # 7. 职业推荐匹配
    career_matches = match_careers(
        archetype_config["archetype_id"],
        archetype_config["archetype_name"],
        riasec_code,
        ocean_scores,
        input_data.career_config,
    )

    # 8. 构建免费卡片数据
    free_card_data = _build_free_card(
        archetype_config, riasec_code, color_spectrum, ocean_scores,
    )

    # 9. 构建输出
    return ScoringEngineOutput(
        archetype_id=archetype_config["archetype_id"],
        archetype_name=archetype_config["archetype_name"],
        archetype_slogan=archetype_config["archetype_slogan"],
        archetype_code=archetype_config["archetype_code"],
        riasec_code=riasec_code,
        color_spectrum=color_spectrum,
        ocean_scores=ocean_scores,
        riasec_scores=riasec_scores,
        free_card_data=free_card_data,
        validity=validity,
        confidence=confidence,
        career_matches=career_matches,
        deep_report_outline=_build_report_outline(
            archetype_config, ocean_scores, riasec_code, career_matches,
        ),
        computed_at=_now_iso(),
    )
```

### 6.4 Redis 分布式锁（支付回调幂等）

```python
# payment/services/callback_handler.py

@asynccontextmanager
async def distributed_lock(redis, lock_key: str, timeout: int = 10):
    """
    Redis 分布式锁，用于支付回调幂等处理。
    确保同一 order_id 的回调串行处理，避免并发导致重复解锁。
    """
    lock_value = f"{order_id}:{uuid4()}"
    acquired = await redis.set(lock_key, lock_value, nx=True, ex=timeout)
    if not acquired:
        raise LockAcquisitionError(f"Lock failed for {lock_key}")
    try:
        yield
    finally:
        lua = """
        if redis.call("get", KEYS[1]) == ARGV[1] then
            return redis.call("del", KEYS[1])
        else
            return 0
        end
        """
        await redis.eval(lua, 1, lock_key, lock_value)

async def handle_payment_callback(order_id: str, callback_data: dict):
    """
    支付回调处理：
    1. 验证签名
    2. 分布式锁保护
    3. 幂等检查
    4. 金额一致性校验
    5. 更新订单状态
    6. 解锁深度报告（切换 is_paid=True，返回完整 12 章深度报告 + 职业推荐）
    """
    lock_key = f"payment:callback:lock:{order_id}"
    try:
        async with distributed_lock(redis, lock_key, timeout=30):
            result = _process_callback(order_id, callback_data)
            return result
    except LockAcquisitionError:
        return CallbackResult(success=True, duplicated=True)
```

## 附录 A：7 点李克特量表说明

本测评全部 80 题采用 7 点李克特量表（Likert Scale），用户针对每条陈述句选择符合程度：

| 量表值 | 标签 | 含义 |
|--------|------|------|
| 1 | 完全不符合 | 该陈述与我的实际情况完全不符 |
| 2 | 比较不符合 | 该陈述与我的实际情况不太相符 |
| 3 | 有些不符合 | 该陈述与我的实际情况有些不符 |
| 4 | 不确定 | 该陈述与我的实际情况部分相符，或难以判断 |
| 5 | 有些符合 | 该陈述与我的实际情况有些相符 |
| 6 | 比较符合 | 该陈述与我的实际情况基本相符 |
| 7 | 非常符合 | 该陈述与我的实际情况完全相符 |

**计分规则**：

| 题目类型 | 计分方式 | 适用范围 |
|---------|---------|---------|
| 正向题 | 直接取量表值（1-7） | RIASEC 全部 30 题 + 大五人格正向题 |
| 反向题 | 翻转计分（1↔7, 2↔6, 3↔5, 4↔4） | 大五人格每维度 2-3 道反向题 |

**反向计分翻转表**：

| 原始选择 | 翻转后得分 | 说明 |
|---------|----------|------|
| 1（完全不符合） | 7 | 不同意反向陈述 = 同意正向特质 |
| 2（比较不符合） | 6 | 基本不同意反向陈述 |
| 3（有些不符合） | 5 | 有些不同意反向陈述 |
| 4（不确定） | 4 | 中立不变 |
| 5（有些符合） | 3 | 有些同意反向陈述 |
| 6（比较符合） | 2 | 基本同意反向陈述 |
| 7（非常符合） | 1 | 完全同意反向陈述 = 不同意正向特质 |

**设计依据**：7 点李克特量表是人格心理学广泛使用的量表格式，相比 5 点量表提供更精细的区分度。相比强制选择量表，李克特量表允许表达中间态度，数据连续性更好，适合百分位标准化与大五人格连续分数模型。
