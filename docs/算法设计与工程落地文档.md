## 1. 🧠 测评模型与多段式编码定义

### 1.1 模型总体架构

采用 **MBTI 单量表为主体 + 两个扩展维度** 的一体化测评方案。全部 86 题在同一量表中完成，扩展维度通过独立题目子集计分，不引入第二个或第三个独立量表。

```
┌──────────────────────────────────────────────────────────┐
│           多段式人格编码 [Base4]-[Sub1]-[Sub2]              │
│                                                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│  │  Base4   │  │  Sub1    │  │  Sub2    │               │
│  │ 如 INTJ  │--│  如 -A   │--│  如 -C   │               │
│  │ 免费可见  │  │ 免费可见  │  │ 免费可见  │               │
│  └──────────┘  └──────────┘  └──────────┘               │
│  4 维 × 18 题   4 题           4 题                        │
│  = 72 题        + 6 效度题 = 86 题总计                    │
│                                                           │
│  ═══════════════════════════════════════════════════════  │
│  免费层：完整编码 INTJ-A-C + 人格认证卡（全段可见）         │
│  付费层：12 章深度报告（职业/压力/成长等维度展开）          │
└──────────────────────────────────────────────────────────┘
```

### 1.2 基础段 (Base4) — MBTI 四维八极

| 维度 | 极 A | 极 B | 测量目标 | 题目数 |
|------|------|------|---------|--------|
| 能量方向 (EI) | E - 外倾 | I - 内倾 | 能量获取方式与社交偏好 | 18 |
| 信息获取 (SN) | S - 感觉 | N - 直觉 | 信息接收与处理偏好 | 18 |
| 决策方式 (TF) | T - 思考 | F - 情感 | 判断与决策依据 | 18 |
| 生活方式 (JP) | J - 判断 | P - 知觉 | 生活组织与执行风格 | 18 |

**商业价值**：16 种类型是完整多段式编码的基础段，与 Sub1、Sub2 共同组成免费人格认证卡（如 `INTJ-A-C`），驱动社交裂变。

### 1.3 附加段 1 (Sub1) — 应对效能维度 `-A` / `-T`

测量个体在压力情境下的自信水平、情绪稳定性与自我效能感。灵感来源于大五人格中的神经质 (Neuroticism) 维度，但简化为二极模型以适配多段式编码。

| 字母 | 名称 | 含义 | 心理学依据 |
|------|------|------|-----------|
| `-A` | Assured 自信型 | 高自我效能、抗压能力强、情绪恢复快、决策果断 | 低神经质 + 高自我效能感 |
| `-T` | Tentative 谨慎型 | 高自我反思、情绪敏感、对反馈响应强、决策审慎 | 中高神经质 + 反刍思维倾向 |

**计分方向**：分数越高 → `-A`；分数越低 → `-T`。

**商业价值**：作为完整多段式编码的第二段（如 `-A`），在免费人格认证卡上与 Base4、Sub2 共同展示。其深度解读内容影响人格优势/劣势的表达风格与成长建议方向，对应深度报告第 5-7 章（人格优势、人格劣势、成长建议）及第 9 章（人格恋爱专题），属于付费层内容。

**题目设计示例**：

| 题号 | 陈述 A (→A 方向) | 陈述 B (→T 方向) |
|------|-----------------|-----------------|
| Q073 | 面对突发困难，我通常能快速冷静下来寻找对策 | 面对突发困难，我容易陷入焦虑和反复担忧 |
| Q074 | 我对自己的能力有信心，很少怀疑自己的决定 | 我经常反思自己是否做了正确的决定 |
| Q075 | 别人的批评不太影响我的情绪状态 | 别人的批评会让我长时间反复琢磨 |
| Q076 | 即使事情进展不顺利，我也能保持积极心态 | 事情进展不顺利时，我容易自我归因 |

### 1.4 附加段 2 (Sub2) — 职场驱动维度 `-C` / `-L`

测量个体的职业价值取向、成就动机与工作生活偏好。融合了麦克利兰成就动机理论与霍兰德职业兴趣理论的核心概念。

| 字母 | 名称 | 含义 | 心理学依据 |
|------|------|------|-----------|
| `-C` | Career-driven 事业型 | 成就导向、竞争意识强、追求职位晋升与社会认可 | 高成就动机 + 企业型 (E) 倾向 |
| `-L` | Lifestyle-driven 生活型 | 成长导向、追求工作生活平衡、重视自我实现与多元体验 | 自主性动机 + 艺术型 (A) / 社会型 (S) 倾向 |

**计分方向**：分数越高 → `-C`；分数越低 → `-L`。

**商业价值**：作为完整多段式编码的第三段（如 `-C`），在免费人格认证卡上与 Base4、Sub1 共同展示。其深度解读内容影响职业方向与岗位推荐，对应深度报告第 11-12 章（深度职业专题、合适的职业），属于付费层内容。

**题目设计示例**：

| 题号 | 陈述 A (→C 方向) | 陈述 B (→L 方向) |
|------|-----------------|-----------------|
| Q077 | 我希望在工作中不断挑战更高目标，获得晋升 | 我希望工作能给我留出足够空间发展个人兴趣 |
| Q078 | 竞争让我兴奋，我享受超越他人的成就感 | 合作让我舒适，我更看重团队的和谐氛围 |
| Q079 | 职业成就和社会认可对我来说非常重要 | 内心充实和生活体验对我来说更重要 |
| Q080 | 我愿意为事业突破牺牲部分个人时间 | 我认为工作与生活的边界应该清晰 |

### 1.5 完整编码空间

16 (Base4) × 2 (Sub1) × 2 (Sub2) = **64 种独特人格编码**

示例编码：

| 编码 | 类型名称 | 解读摘要 | 可见层级 |
|------|---------|---------|---------|
| `INTJ-A-C` | 自信事业型建筑师 | 战略思维 + 高抗压 + 成就驱动 | 免费可见（编码+卡片）|
| `INTJ-T-L` | 谨慎生活型建筑师 | 战略思维 + 深度反思 + 平衡导向 | 免费可见（编码+卡片）|
| `ENFP-A-L` | 自信生活型竞选者 | 热情创变 + 积极乐观 + 多元探索 | 免费可见（编码+卡片）|

> 全部 64 种编码均在免费层完整可见。付费层解锁的是基于编码生成的 12 章深度报告。

### 1.7 免费人格认证卡设计规范（基于 `simple.png` 参考稿）

## 2. 🗄️ 核心数据结构设计 (Data Schema)

### 2.1 Pydantic V2 Schema 定义

全部数据结构使用 Pydantic V2，**零 Django/FastAPI 依赖**。

```python
# scoring/schemas.py
from __future__ import annotations
from enum import Enum
from pydantic import BaseModel, Field, field_validator
from typing import Literal


# ========== 枚举定义 ==========

class BaseDimension(str, Enum):
    EI = "EI"
    SN = "SN"
    TF = "TF"
    JP = "JP"

class ExtendedDimension(str, Enum):
    """扩展维度（附加段）"""
    AT = "AT"   # A/T 应对效能
    CL = "CL"   # C/L 职场驱动

class BasePole(str, Enum):
    E = "E"; I = "I"
    S = "S"; N = "N"
    T = "T"; F = "F"
    J = "J"; P = "P"

class Sub1Pole(str, Enum):
    A = "A"   # Assured 自信型
    T = "T"   # Tentative 谨慎型

class Sub2Pole(str, Enum):
    C = "C"   # Career-driven 事业型
    L = "L"   # Lifestyle-driven 生活型

class QuestionType(str, Enum):
    NORMAL = "normal"
    REVERSE = "reverse"
    LIE_SCALE = "lie_scale"
    CONSISTENCY = "consistency"

class ScalePoint(int, Enum):
    """6 点量表强制选择，无中性中点"""
    STRONG_AGREE_A = 1
    AGREE_A = 2
    SLIGHT_A = 3
    SLIGHT_B = 4
    AGREE_B = 5
    STRONG_AGREE_B = 6

class DimensionGroup(str, Enum):
    """题目所属维度组"""
    BASE = "base"          # 基础段 (EI/SN/TF/JP)
    SUB1 = "sub1"          # 附加段1 (A/T)
    SUB2 = "sub2"          # 附加段2 (C/L)
    VALIDITY = "validity"  # 效度题


# ========== 题目与选项 ==========

class Option(BaseModel):
    """选项元数据"""
    scale_value: ScalePoint
    pole: str                           # 该选项倾向的极字母 (E/I/S/N/T/F/J/P/A/T/C/L)
    raw_score: int = Field(ge=1, le=3)  # 转换后的原始分 (1-3)

class Question(BaseModel):
    """题目结构"""
    question_id: str = Field(pattern=r"^Q\d{3}$")
    dimension_group: DimensionGroup      # 所属维度组
    base_dimension: BaseDimension | None = None   # 基础段题目所属维度
    extended_dimension: ExtendedDimension | None = None  # 扩展段题目所属维度
    question_type: QuestionType
    statement_a: str
    statement_b: str
    pole_a: str                          # 陈述A对应的极字母
    pole_b: str                          # 陈述B对应的极字母
    pair_id: str | None = None           # 一致性检测题配对ID
    is_active: bool = True


# ========== 用户答题流水 ==========

class UserAnswerLog(BaseModel):
    """单题答题记录（敏感数据，需加密存储）"""
    question_id: str
    scale_value: ScalePoint
    response_duration_ms: int = Field(ge=0)
    modification_count: int = Field(ge=0, default=0)
    answered_at: str                     # ISO 8601

class AssessmentSubmission(BaseModel):
    """用户完整提交"""
    session_token: str
    device_fingerprint: str
    answers: list[UserAnswerLog]
    started_at: str
    submitted_at: str

    @field_validator("answers")
    @classmethod
    def validate_answer_count(cls, v: list[UserAnswerLog]) -> list[UserAnswerLog]:
        if len(v) != 86:
            raise ValueError(f"Expected 86 answers, got {len(v)}")
        return v


# ========== 维度得分 ==========

class BaseDimensionScore(BaseModel):
    """基础段维度得分"""
    dimension: BaseDimension
    raw_score: int                       # -54 ~ +54 (18题 × 3分)
    pole_a_score: int                    # 0-54
    pole_b_score: int                    # 0-54
    pole_a_percent: float                # 0-100
    pole_b_percent: float                # 0-100
    dominant_pole: BasePole
    clarity: float                       # 0-100 偏好清晰度
    t_score: float                       # 标准分
    percentile: int                      # 百分位

class ExtendedDimensionScore(BaseModel):
    """扩展段维度得分"""
    dimension: ExtendedDimension
    raw_score: int                       # -12 ~ +12 (4题 × 3分)
    pole_a_score: int                    # 0-12
    pole_b_score: int                    # 0-12
    pole_a_percent: float                # 0-100
    pole_b_percent: float                # 0-100
    dominant_pole: str                   # Sub1Pole 或 Sub2Pole 的值
    clarity: float                       # 0-100
    t_score: float
    percentile: int


# ========== 效度结果 ==========

class ValidityResult(BaseModel):
    is_valid: bool
    invalid_reasons: list[str] = Field(default_factory=list)
    confidence_score: float = Field(ge=0.0, le=1.0)
    lie_scale_triggered: bool = False
    response_time_anomaly: bool = False
    straight_lining_detected: bool = False
    consistency_failed: bool = False


# ========== 职业推荐 ==========

class CareerMatch(BaseModel):
    career_id: str
    career_name: str
```

### 3.1 6 点量表 → 方向分转换

```python
SCALE_TO_DIRECTIONAL: dict[int, tuple[str, int]] = {
    1: ("A", 3),   # 非常同意 A → +3
    2: ("A", 2),   # 比较同意 A → +2
    3: ("A", 1),   # 稍微同意 A → +1
    4: ("B", 1),   # 稍微同意 B → +1
    5: ("B", 2),   # 比较同意 B → +2
    6: ("B", 3),   # 非常同意 B → +3
```

### 3.2 基础段原始分计算

```python
# scoring/calculators.py

def calculate_base_raw_score(
    answers: list[UserAnswerLog],
    config: DimensionConfig,
) -> tuple[int, int, int]:
    """
    基础段维度原始分计算。
    
    每维度 18 题，满分 54 分/极。
    raw_score = pole_a_total - pole_b_total ∈ [-54, +54]
    raw_score > 0 → pole_a 占优
    
    支持反向题：reverse_question_ids 中的题目交换 A/B 方向。
    """
    pole_a_total = 0
    pole_b_total = 0
    reverse_ids = set(config.reverse_question_ids)

    for q_id in config.question_ids:
        answer = _find_answer(answers, q_id)
        direction, weight = SCALE_TO_DIRECTIONAL[answer.scale_value]

        if q_id in reverse_ids:
            direction = "B" if direction == "A" else "A"

        if direction == "A":
            pole_a_total += weight
        else:
            pole_b_total += weight

    raw_score = pole_a_total - pole_b_total
    return raw_score, pole_a_total, pole_b_total
```

**数学公式**（基础段每维度 $N=18$ 题）：

$$\text{raw\_score}_D = \sum_{i=1}^{N} s_i \times (-1)^{r_i}, \quad s_i \in \{-3,-2,-1,+1,+2,+3\}$$

取值范围 $[-3N, +3N] = [-54, +54]$。

### 3.3 扩展段原始分计算

扩展段使用完全相同的计分逻辑，但题目数为 4，满分 12 分/极：

```python
def calculate_extended_raw_score(
    answers: list[UserAnswerLog],
    config: ExtendedDimensionConfig,
) -> tuple[int, int, int]:
    """
    扩展段维度原始分计算。
    
    每维度 4 题，满分 12 分/极。
    raw_score ∈ [-12, +12]
    raw_score > 0 → pole_a 占优 (A 或 C)
    raw_score < 0 → pole_b 占优 (T 或 L)
    """
    # 逻辑与 base 完全一致，仅 question_count 不同
    return calculate_base_raw_score(answers, config)
```

### 3.4 标准化转换

#### 百分比转换

$$\text{pole\_a\_percent} = 50 + \frac{\text{raw\_score}}{6N} \times 100$$

基础段：$N=18$，范围 $[0, 100]$；扩展段：$N=4$，范围 $[0, 100]$。

#### T 分数转换

$$T = 50 + 10 \times \frac{\text{raw\_score} - \mu_D}{\sigma_D}$$

$\mu_D$、$\sigma_D$ 从配置文件加载（基于中国用户常模）。

#### 百分位排名

```python
from scipy.stats import norm

def raw_to_percentile(raw_score: int, norm_mean: float, norm_std: float) -> int:
    z = (raw_score - norm_mean) / norm_std
    return int(round(norm.cdf(z) * 100))
```

#### 偏好清晰度

$$\text{clarity} = \frac{|\text{raw\_score}|}{3N} \times 100$$

```python
# scoring/engine.py

def generate_personality_code(
    base_dimensions: list[BaseDimensionScore],
    sub1_dimension: ExtendedDimensionScore,
    sub2_dimension: ExtendedDimensionScore,
    config: ScoringConfig,
) -> tuple[str, str, str, str]:
    """
    生成多段式人格编码。
    
    Returns:
        (base_code, sub1_code, sub2_code, full_code)
        如 ("INTJ", "A", "C", "INTJ-A-C")
    """

    # ========== 1. 基础段：4 字母生成 ==========
    base_letters: list[str] = []
    for dim_score in base_dimensions:
        letter = _determine_pole_letter(
            dim_score.pole_a_percent,
            dim_score.dominant_pole,
            config.base_dimensions[dim_score.dimension],
            config.base_tie_breaker,
        )
        base_letters.append(letter)
    base_code = "".join(base_letters)  # 如 "INTJ"

    # ========== 2. 附加段 1：A/T 判定 ==========
    sub1_code = _determine_extended_letter(
        sub1_dimension,
        config.extended_dimensions["AT"],
        config.extended_tie_breaker,
    )  # "A" 或 "T"

    # ========== 3. 附加段 2：C/L 判定 ==========
    sub2_code = _determine_extended_letter(
        sub2_dimension,
        config.extended_dimensions["CL"],
        config.extended_tie_breaker,
    )  # "C" 或 "L"

    # ========== 4. 拼接完整编码 ==========
    full_code = f"{base_code}-{sub1_code}-{sub2_code}"  # 如 "INTJ-A-C"

    return base_code, sub1_code, sub2_code, full_code


def _determine_pole_letter(
    pole_a_percent: float,
    dominant_pole: str,
    dim_config: DimensionConfig,
    tie_breaker: str,
) -> str:
    """基础段单维度字母判定"""
    if pole_a_percent > 50.0:
        return dim_config.pole_a   # 如 "I"
    elif pole_a_percent < 50.0:
        return dim_config.pole_b   # 如 "E"
    else:
        # 平局处理：取常模均值方向
        if dim_config.norm_mean >= 0:
            return dim_config.pole_a
        else:
            return dim_config.pole_b


def _determine_extended_letter(
    dim_score: ExtendedDimensionScore,
    dim_config: ExtendedDimensionConfig,
    tie_breaker: str,
) -> str:
    """扩展段字母判定（逻辑与基础段一致）"""
    return _determine_pole_letter(
        dim_score.pole_a_percent,
        dim_score.dominant_pole,
        dim_config,
        tie_breaker,
    )
```

### 3.7 职业推荐匹配算法

职业匹配考虑基础段类型 + 附加段偏好：

```python
def match_careers(
    full_code: str,
    base_code: str,
    sub1_code: str,
    sub2_code: str,
    base_dimensions: list[BaseDimensionScore],
    career_config: dict,
) -> list[CareerMatch]:
    """
    多段式编码驱动的职业推荐。
    
    匹配公式:
    match_score = base_affinity (0-60)
               + sub2_bonus (0-25)      # 职场驱动维度影响最大
               + sub1_bonus (0-10)      # 应对效能影响适配角匹配度
               + clarity_bonus (0-5)    # 维度清晰度加成
    """
    matches: list[CareerMatch] = []

    for career_id, career_data in career_config.items():
        # 基础亲和度：类型与职业的基础匹配
        base_affinity = career_data["type_affinity"].get(base_code, 30)

        # Sub2 加成：职场驱动与职业特性匹配
        career_drive = career_data.get("drive_type", "balanced")
        sub2_bonus = _calc_sub2_bonus(sub2_code, career_drive)

        # Sub1 加成：应对效能与职业压力水平匹配
        career_stress = career_data.get("stress_level", "medium")
        sub1_bonus = _calc_sub1_bonus(sub1_code, career_stress)

        # 清晰度加成
        avg_clarity = sum(d.clarity for d in base_dimensions) / len(base_dimensions)
        clarity_bonus = min(5, avg_clarity / 20)

        total = base_affinity + sub2_bonus + sub1_bonus + clarity_bonus
        total = min(100, total)

        match_level = "high" if total >= 75 else "medium" if total >= 50 else "low"

        matches.append(CareerMatch(
            career_id=career_id,
            career_name=career_data["name"],
            match_score=round(total, 1),
            match_level=match_level,
            tags=career_data.get("tags", []),
            recommended_for_code=full_code,
        ))

    matches.sort(key=lambda m: m.match_score, reverse=True)
    return matches[:10]  # 返回 Top 10


def _calc_sub2_bonus(sub2_code: str, career_drive: str) -> int:
```

### 4.2 矛盾题对检测

```python
def check_consistency(
    answers: list[UserAnswerLog],
    consistency_pairs: list[ConsistencyPair],
) -> bool:
    """
    矛盾题对一致性检测。
    
    逻辑：配对题的量表值应在对称位置（Q003 选 1 ↔ Q014 应选 6）。
    若两题方向一致（都选 A 或都选 B），计为矛盾。
    
    判定：3 对中 >= 2 对矛盾 → consistency_failed = True
    """
    conflict_count = 0

    for pair in consistency_pairs:
        ans_a = _find_answer(answers, pair.item_a)
        ans_b = _find_answer(answers, pair.item_b)

        dir_a = "A" if ans_a.scale_value <= 3 else "B"
        dir_b = "A" if ans_b.scale_value <= 3 else "B"

        if dir_a == dir_b:
            conflict_count += 1

    return conflict_count >= 2
```

### 4.3 行为异常检测

```python
def detect_behavior_anomaly(answers: list[UserAnswerLog]) -> BehaviorAnomaly:
    durations = [a.response_duration_ms for a in answers]
    values = [a.scale_value for a in answers]

    # 1. 响应过快：平均耗时 < 1500ms
    avg_duration = sum(durations) / len(durations)
    too_fast = avg_duration < 1500

    # 2. 直线作答：同一选项占比 > 80%
    from collections import Counter
    value_counts = Counter(values)
    max_ratio = max(value_counts.values()) / len(values)
    straight_lining = max_ratio > 0.80

    # 3. 交替模式检测：如 1,6,1,6,... 连续 10+ 题
    alternating = _detect_alternating_pattern(values)

    # 4. 响应时间方差过低：std < 500ms（自动化工具特征）
    import statistics
    duration_std = statistics.stdev(durations) if len(durations) > 1 else 0
    too_uniform = duration_std < 500

    return BehaviorAnomaly(
        too_fast=too_fast,
        straight_lining=straight_lining,
        alternating=alternating,
        too_uniform=too_uniform,
```

| 响应时间标准差 | < 500 ms | `response_time_anomaly = True` |
| 测谎题得分 | >= 2/3 | `lie_scale_triggered = True` |
| 矛盾题对冲突 | >= 2/3 | `consistency_failed = True` |

### 4.4 效度标记与置信度计算

```python
def compute_validity(
    answers: list[UserAnswerLog],
    config: ValidityConfig,
) -> ValidityResult:
    lie_triggered = check_lie_scale(answers, config.lie_scale_ids)
    consistency_failed = check_consistency(answers, config.consistency_pairs)
    anomaly = detect_behavior_anomaly(answers)

    invalid_reasons: list[str] = []
    penalty = 0.0

    if lie_triggered:
        invalid_reasons.append("lie_scale_triggered")
        penalty += 0.3
    if consistency_failed:
        invalid_reasons.append("consistency_failed")
        penalty += 0.25
    if anomaly.too_fast or anomaly.too_uniform or anomaly.alternating:
        invalid_reasons.append("response_time_anomaly")
        penalty += 0.2
    if anomaly.straight_lining:
        invalid_reasons.append("straight_lining_detected")
        penalty += 0.15

    is_valid = len(invalid_reasons) == 0
    confidence = max(0.0, 1.0 - penalty)

    return ValidityResult(
        is_valid=is_valid,
        invalid_reasons=invalid_reasons,
        confidence_score=confidence,
        lie_scale_triggered=lie_triggered,
        response_time_anomaly=anomaly.too_fast or anomaly.too_uniform,
        straight_lining_detected=anomaly.straight_lining,
        consistency_failed=consistency_failed,
    )
```

**效度结果处理策略**：

| `is_valid` | `confidence_score` | 处理策略 |
|-----------|-------------------|---------|
| True | >= 0.8 | 正常出报告 |
| True | 0.5 - 0.8 | 出报告但标注"结果仅供参考" |
| False | < 0.5 | 提示"测评效度不足，建议重新测试"，不出报告 |

```python
# scoring/engine.py

class ScoringEngineInput(BaseModel):
    """计分引擎输入（纯数据，无框架依赖）"""
    answers: list[UserAnswerLog]         # 86 条答题记录
    question_bank_config: dict           # 题库配置

    type_configs: dict                   # 16 种类型 × 2 Sub1 × 2 Sub2 配置
    validity_config: dict                # 效度检测配置
    career_config: dict                  # 职业推荐配置

class ScoringEngineOutput(BaseModel):
    """计分引擎输出（纯数据，无框架依赖）"""
    # 基础段（免费可见）
    base_code: str                       # "INTJ"
    base_type_name: str                  # "建筑师"
    base_type_slogan: str                # "想象一切，掌控未来"
    base_dimensions: list[BaseDimensionScore]
    base_radar_data: dict
    # 附加段（免费可见）
    sub1_code: str                       # "A"
    sub1_name: str                       # "自信型"
    sub1_dimension: ExtendedDimensionScore
    sub2_code: str                       # "C"
    sub2_name: str                       # "事业型"
    sub2_dimension: ExtendedDimensionScore
    # 完整编码（免费可见）
    full_code: str                       # "INTJ-A-C"
    # 免费卡片数据（免费可见，参照 simple.png）
    free_card_data: dict
    # 付费层内容
    validity: ValidityResult
    career_matches: list[CareerMatch]    # 付费
    deep_report_outline: dict            # 付费
    computed_at: str


def calculate_personality_result(
    input_data: ScoringEngineInput,
) -> ScoringEngineOutput:
    """
    多段式人格编码计分引擎唯一入口函数。
    
    特性：
    - 纯函数：相同输入 → 相同输出（计分确定性）
    - 无副作用：不访问数据库、不调用网络、不写文件
    - 无框架依赖：不导入 Django / FastAPI 任何模块
    - 全配置驱动：所有规则从 input_data 中的 config 读取
    
    流程：
    1. 输入校验（86 题完整性）
    2. 效度检测（测谎题 + 矛盾题 + 行为异常）
    3. 基础段计分（4 维度 × 18 题 → 4 字母）
    4. 扩展段计分（2 维度 × 4 题 → 2 字母）
    5. 多段式编码拼接（INTJ-A-C）
    6. 职业推荐匹配
    7. 报告大纲生成
    """
    # 1. 输入校验
    _validate_answers(input_data.answers, input_data.question_bank_config)

    # 2. 效度检测
    validity = compute_validity(input_data.answers, input_data.validity_config)

    # 3. 基础段计分
    base_dimensions: list[BaseDimensionScore] = []
    for dim in [BaseDimension.EI, BaseDimension.SN, BaseDimension.TF, BaseDimension.JP]:
        dim_config = input_data.scoring_config["base_dimensions"][dim.value]
        raw, pa, pb = calculate_base_raw_score(input_data.answers, dim_config)
        dim_score = _build_base_dimension_score(dim, raw, pa, pb, dim_config)
        base_dimensions.append(dim_score)

    # 4. 扩展段计分
    at_config = input_data.scoring_config["extended_dimensions"]["AT"]
    at_raw, at_pa, at_pb = calculate_extended_raw_score(input_data.answers, at_config)
    sub1_dimension = _build_extended_score(ExtendedDimension.AT, at_raw, at_pa, at_pb, at_config)

    cl_config = input_data.scoring_config["extended_dimensions"]["CL"]
    cl_raw, cl_pa, cl_pb = calculate_extended_raw_score(input_data.answers, cl_config)
    sub2_dimension = _build_extended_score(ExtendedDimension.CL, cl_raw, cl_pa, cl_pb, cl_config)

    # 5. 多段式编码生成
    base_code, sub1_code, sub2_code, full_code = generate_personality_code(
        base_dimensions, sub1_dimension, sub2_dimension, input_data.scoring_config
    )

    # 6. 类型配置查询
    type_config = input_data.type_configs[base_code]

    # 7. 职业推荐
    career_matches = match_careers(
        full_code, base_code, sub1_code, sub2_code,
        base_dimensions, input_data.career_config
    )

    # 8. 构建输出
    return ScoringEngineOutput(
        base_code=base_code,
        base_type_name=type_config["base"]["type_name"],
        base_type_slogan=type_config["base"]["type_slogan"],
        base_dimensions=base_dimensions,
        base_radar_data=_build_radar_data(base_dimensions),
        free_card_data=_build_free_card(type_config, base_dimensions),
        sub1_code=sub1_code,
        sub1_name=at_config[f"pole_{sub1_code.lower()}_name"],
        sub1_dimension=sub1_dimension,
        sub2_code=sub2_code,
        sub2_name=cl_config[f"pole_{sub2_code.lower()}_name"],
        sub2_dimension=sub2_dimension,
        full_code=full_code,
        validity=validity,
        career_matches=career_matches,
        deep_report_outline=_build_report_outline(type_config, base_dimensions, sub1_code, sub2_code),
        computed_at=_now_iso(),
    )
```

### 6.4 Redis 分布式锁（支付回调幂等）

```python
# payment/services/callback_handler.py

@asynccontextmanager
async def distributed_lock(redis, lock_key: str, timeout: int = 10):
    """
    Redis 分布式锁，用于支付回调幂等处理。
    确保同一 order_id 的回调串行处理，避免并发导致重复解锁。
    """
    lock_value = f"{order_id}:{uuid4()}"
    acquired = await redis.set(lock_key, lock_value, nx=True, ex=timeout)
    if not acquired:
        raise LockAcquisitionError(f"Lock failed for {lock_key}")
    try:
        yield
    finally:
        lua = """
        if redis.call("get", KEYS[1]) == ARGV[1] then
            return redis.call("del", KEYS[1])
        else
            return 0
        end
        """
        await redis.eval(lua, 1, lock_key, lock_value)

async def handle_payment_callback(order_id: str, callback_data: dict):
    """
    支付回调处理：
    1. 验证签名
    2. 分布式锁保护
    3. 幂等检查
    4. 金额一致性校验
    5. 更新订单状态
    6. 解锁深度报告（切换 is_paid=True，返回完整 12 章深度报告 + 职业推荐）
    """
    lock_key = f"payment:callback:lock:{order_id}"
    try:
        async with distributed_lock(redis, lock_key, timeout=30):
            result = _process_callback(order_id, callback_data)
            return result
    except LockAcquisitionError:
        return CallbackResult(success=True, duplicated=True)
```
