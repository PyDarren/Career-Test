# 职业测评 Web 应用 — 项目排期表

> **文档版本**：V1.0 | **日期**：2026-07-11
>
> **依据文档**：`PRD.md` V1.1、`AGENTS.md`、`算法设计与工程落地文档.md` V2.1
>
> **MVP 目标**：6 个月内完成上线，验证付费转化率 ≥ 8%

---

## 1. 项目概况

### 1.1 核心交付物

| 维度 | 数量 | 说明 |
|------|------|------|
| Django App 模块 | 5 个 | assessment / mbti_types / careers / payment / stats |
| 数据库表 | 9 张 | 禁止新增或删除 |
| API 端点 | 19 个 | 以 `/api/` 为前缀，RESTful 风格 |
| Celery 定时任务 | 5 个 | 订单过期 / 数据清理 / 日报 / 缓存刷新 / 对账 |
| 前端页面 | 10 个 | Django 模板渲染，16personalities.com 风格 |
| 深度报告章节 | 12 章 | 以 `人格测试报告.md` 为权威模板 |
| 测评题目 | 86 题 | 6 点量表强制选择，单量表架构 |
| 人格编码 | 64 种 | 16(Base4) × 2(Sub1) × 2(Sub2) |
| 吉祥物图片 | 16 种 | 3D 黏土风格，按 Base4 类型匹配 |

### 1.2 技术栈

Python 3.11+ / Django 5.0+ / FastAPI 0.110+ / MySQL 8.0 / Redis 7.0 / Celery 5.3 / Nginx → Gunicorn → Django → MySQL + Redis

### 1.3 团队角色

| 角色 | 职责范围 |
|------|---------|
| 后端 Agent | Django apps（models/views/services/urls/middleware）、Celery tasks、FastAPI 辅助接口 |
| 前端 Agent | Django 模板（HTML）、static/js/ 交互逻辑、static/css/ 样式、微信 JS-SDK |
| 算法/内容 Agent | assessment/scoring/ 计分引擎、深度报告 12 章内容、MBTI 类型配置数据、初始数据 fixture |

## 6. 关键风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 计分引擎边界值处理不当 | 用户类型判定错误 | 2A.9 单元测试覆盖 ≥ 95%，含全选A/全选B/平局/极端比例 |
| 支付回调被伪造或重放 | 资金损失 | 六道防线逐项实现 + 3A.9 安全测试 |
| 微信 JS-SDK 权限配置错误 | 分享/支付失败 | 3B.5 提前联调，jsapi_ticket 缓存 TTL 7200 秒 |
| 题库数据质量不达标 | 测评信效度低 | 2A.2 配置化 JSON + 效度题检测 + 算法 Agent 评审 |
| 深度报告内容不全 | 付费转化率低 | 2A.13 以 `人格测试报告.md` 为模板，16 类型 × 12 章提前生成 |
| 高并发下性能瓶颈 | 页面加载慢 | 4.9 性能优化 + Redis 缓存 + select_related / prefetch_related |
| 吉祥物图片缺失 | 卡片视觉不完整 | 4.14 16 种吉祥物提前安排设计，M4 前交付 |
| 敏感数据泄露 | 合规风险 | 4.13 安全审计 + AES-256 加密 + 日志脱敏 + .env 不入 Git |
