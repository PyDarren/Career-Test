## 9. 关键风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 文档不一致导致开发偏差 | AI 助手依据过时 AGENTS.md 生成 MBTI 代码 | M0 阶段强制完成所有文档对齐，M1 开始前确认无 MBTI 残留 |
| 计分引擎边界值处理不当 | 画像匹配错误 | 2A.12 单元测试覆盖 ≥ 95%，含全选1/全选5/平局/极端比例/反向题验证 |
| RIASEC 并列分数处理不当 | 兴趣码生成不稳定 | 2A.7 明确并列时按 R>I>A>S>E>C 字母顺序优先 |
| 支付回调被伪造或重放 | 资金损失 | 六道防线逐项实现 + 3A.9 安全测试 |
| 微信 JS-SDK 权限配置错误 | 分享/支付失败 | 3B.5 提前联调，jsapi_ticket 缓存 TTL 7200 秒 |
| 题库数据质量不达标 | 测评信效度低 | 2A.2+2A.3 配置化 JSON + 效度题检测 + 算法 Agent 评审 |
| 32 画像 × 12 章报告内容不全 | 付费转化率低 | 2A.16 以重写后的报告模板为准，提前生成 384 章内容 |
| 高并发下性能瓶颈 | 页面加载慢 | 4.10 性能优化 + Redis 缓存 + select_related / prefetch_related |
| 32 种吉祥物图片缺失 | 卡片视觉不完整 | 4.15 提前安排 3D 黏土风格设计，M4 前交付 |
| 敏感数据泄露 | 合规风险 | 4.14 安全审计 + AES-256 加密 + 日志脱敏 + .env 不入 Git |
| IPIP 题项中文化精度不足 | 测评信效度下降 | 参考现有中文翻译版本 + 心理学顾问审校 |
| 后端从零搭建周期紧张 | M1 延期影响后续 | M0 文档对齐与 M1 初始化可部分并行；优先搭建 assessment + personality 两个核心 App |
