# Anchor — 人格原型扩展方案（32 型 → 243 型）

> **版本**：V3.0 | **日期**：2026-07-14
>
> **基础框架**：IPIP 大五人格（OCEAN）五维度三级分制
>
> **扩展原理**：将每维度从二级（高/低）升级为三级（高/中/低），3^5 = 243 种原型
>
> **命名规范**：所有原型名称采用英文大写字母标签格式，由「族群码-修饰词码」两段式组成
>
> **职业推荐**：每种原型均配有 2-3 个匹配职位建议，基于族群行业领域与修饰词工作风格组合生成

## 1. 扩展方法论

### 1.1 为什么扩展

当前 32 型系统采用二级分制（高/低），将每维度仅分为两档。这意味着百分位 51% 和 99% 的用户被归为同一档（高），丢失了大量中间梯度信息。

| 对比维度 | 32 型（现有） | 243 型（扩展） |
|---------|-------------|--------------|
| 分制 | 二级：H/L | 三级：H/M/L |
| 维度切分点 | 第 50 百分位 | 第 33.3 和 66.7 百分位 |
| 组合数 | 2^5 = 32 | 3^5 = 243 |
| 分辨率 | 每 50% 为一档 | 每 33.3% 为一档 |
| 中间型覆盖 | 无（中间用户被强行二分） | 有（M 档捕捉中间特征） |
| 极端型精度 | 低（51% 和 99% 同档） | 高（仅 66.7% 以上为 H） |

### 1.2 扩展原则

- **理论一致**：仍基于 IPIP 大五人格，不引入新理论框架，零侵权风险
- **算法兼容**：计分引擎不变，仅修改 `percentile_to_level()` 函数的阈值
- **向后兼容**：旧 32 型可映射到新 243 型的子集（H→H 或 M，L→M 或 L）
- **命名系统化**：采用「族群码-修饰词码」英文大写两段式标签，可程序化生成

### 1.3 标签命名规则

所有 243 种原型均采用英文大写字母标签，格式为 `XXX-YYY`：

- **族群码（XXX）**：3 个大写英文字母，由 O×C 组合决定（9 种）
- **修饰词码（YYY）**：3 个大写英文字母，由 E×A×N 组合决定（27 种）
- **完整标签**：`族群码-修饰词码`，如 `VSN-CSK`、`INS-FHR`、`CTN-GLC`

标签设计原则：取自该族群/修饰词英文全称的关键字母组合，确保唯一性和可读性。

### 1.4 分制对比示意

```
百分位:  0% ──────── 33.3% ──────── 66.7% ──────── 100%

32型:    ├──── 低 (L) ──────────┤├──── 高 (H) ──────────┤

243型:   ├── 低 (L) --┤├-- 中 (M) --┤├-- 高 (H) --┤
```

## 2. 三级分制详解

### 2.1 切分标准

| 等级 | 百分位范围 | 含义 | 人群占比 |
|------|-----------|------|---------|
| H（高） | 66.7% ~ 100% | 该维度显著高于常模 | 约 33.3% |
| M（中） | 33.3% ~ 66.7% | 该维度接近常模均值 | 约 33.3% |
| L（低） | 0% ~ 33.3% | 该维度显著低于常模 | 约 33.3% |

### 2.2 编码规则

5 字符编码，按 O-C-E-A-N 固定顺序，每字符为 H/M/L：

```
HMMHL
│││││
││││└── N 维度：L = 低神经质（情绪稳定）
│││└─── A 维度：H = 高宜人性
││└──── E 维度：M = 中外向性
│└───── C 维度：M = 中尽责性
└────── O 维度：H = 高开放性
```

### 2.3 计分引擎修改点

仅需修改 `assessment/scoring/normalizer.py` 中的 `percentile_to_level()` 函数：

```python
# 现有（二级分制）
def percentile_to_level(percentile: float) -> int:
    if percentile < 50.0:
        return 1  # L
    return 5      # H

# 扩展后（三级分制）
def percentile_to_level(percentile: float) -> int:
    if percentile < 33.3:
        return 1  # L
    if percentile < 66.7:
        return 3  # M
    return 5      # H
```

`archetype_matcher.py` 中的匹配逻辑从 2^5 查表改为 3^5 查表，编码从 10 字符（OHCHEHAHNL）简化为 5 字符（HMMHL）。

| L（百分位 ≤ 50） | 低 | M | 33.3 < 百分位 ≤ 66.7 |
| L（百分位 ≤ 50） | 低 | L | 百分位 ≤ 33.3 |

### 9.5 生成算法

```python
def get_career_suggestion(family_code: str, E: str, A: str, N: str) -> str:
    # 1. 根据族群码选择行业领域职业池
    family = FAMILY_CAREER_POOLS[family_code]
    # 2. 根据 E 级别选择工作风格子池（EH/EM/EL）
    e_pool = family[f"careers_E{E}"]
    # 3. 根据 A×N 组合选择具体职位
    career = e_pool[(A, N)]
    return career
```

每个族群维护 3 个 E 级别子池，每个子池包含 9 种 A×N 组合的职位，合计 9 × 3 × 9 = 243 种唯一职位建议。
