## 3. 算法与数据测试

### 3.1 测评算法准确性验证提示词

```
你是一名数据测试工程师，负责验证「画己职测」测评算法的准确性。

测评算法基于：
- IPIP 大五人格量表（50 题，OCEAN 五维度，各 10 题）
- 霍兰德 RIASEC 职业兴趣理论（30 题，六类型各 5 题）
- 5 点李克特量表（1=非常不符合，5=非常符合）
- 部分题目为反向计分题（得分 = 6 - 原始分）

请执行以下测试：

【步骤一：构造测试样本】
构造至少 5 组答题样本，每组 80 题答案，覆盖不同人格特征：

样本 1 — 高开放性高外向性：
  - BO 题目全部选 5（正向）/ 选 1（反向）
  - BE 题目全部选 5（正向）/ 选 1（反向）
  - 其余维度交替选 3 和 4

样本 2 — 高尽责性低神经质：
  - BC 题目全部选 5（正向）/ 选 1（反向）
  - BN 题目全部选 1（正向）/ 选 5（反向）
  - 其余维度交替选 3 和 4

样本 3 — 全部居中选 3（直线作答检测）：
  - 全部 80 题选 3
  - 预期：置信度显著降低或标记为无效

样本 4 — 高宜人性高社会型：
  - BA 题目全部选 5（正向）/ 选 1（反向）
  - RS 题目全部选 5（正向）/ 选 1（反向）
  - 其余维度交替选 3 和 4

样本 5 — 随机作答：
  - 每题随机选 1-5
  - 预期：结果有效但置信度可能偏低

【步骤二：手工核算】
对样本 1 和样本 2，手工计算：
1. OCEAN 各维度原始分（注意反向计分：反向题得分 = 6 - 选项值）
2. 各维度百分位（与常模对比）
3. 根据 OCEAN 高低组合确定 32 画像中的哪一个
4. RIASEC 六类型得分，取前三高组成三字母代码
5. 画像稀有度等级

【步骤三：系统对比】
将同样答案通过 API（POST /api/assessments/）提交，获取系统返回结果，与手工核算逐项对比：
- OCEAN 五维度百分位差异 ≤ 2 个百分点
- RIASEC 三字母代码完全一致
- 画像 ID 完全一致
- 置信度差异 ≤ 0.05

【步骤四：边界测试】
1. 全部选 1：预期 OCEAN 全低，某特定画像
2. 全部选 5：预期 OCEAN 全高，某特定画像
3. 交替选 1 和 5：验证反向计分逻辑
4. 仅答前 40 题：预期系统拒绝提交或提示未完成

请输出对比表格：
| 样本 | 维度 | 手工得分 | 系统得分 | 差值 | 是否一致 |
```

## 5. 安全测试

### 5.1 Web 安全测试提示词

```
你是一名安全测试工程师，请对「画己职测」进行 Web 安全审计。

测试范围：http://127.0.0.1:8000

【A. SQL 注入测试】
A1. 在以下输入点尝试 SQL 注入 payload：
   - 答题提交接口 POST /api/assessments/（answers 字段）
   - 订单查询接口 GET /api/orders/{order_id}/status/
   - 后台搜索接口 GET /api/admin/questions/?search=
   - 职业推荐接口 GET /api/careers/?archetype=&riasec=

   测试 payload：
   - ' OR '1'='1
   - 1; DROP TABLE questions; --
   - ' UNION SELECT * FROM auth_user; --
   - 1' AND SLEEP(5); --

   预期：所有注入尝试被 Django ORM 参数化查询拦截，返回正常错误响应或空结果，
   不返回数据库结构信息，不产生延迟。

A2. 验证 Django ORM 参数化查询：
   - 确认所有数据库查询使用 ORM 的 filter()/exclude()，不使用 raw SQL
   - 如有 raw SQL，确认使用参数化占位符

【B. XSS 攻击测试】
B1. 存储型 XSS：
   - 在答题提交的 user_agent 字段注入：<script>alert('xss')</script>
   - 在后台内容配置的文案字段注入：<img src=x onerror=alert(1)>
   - 预期：Django 模板自动转义，脚本不执行

B2. 反射型 XSS：
   - 访问 /api/admin/questions/?search=<script>alert(1)</script>
   - 预期：参数被转义，不执行脚本

B3. DOM 型 XSS：
   - 检查前端 JS 代码中是否有 innerHTML、document.write 等危险操作
   - 检查 URL 参数是否被直接插入 DOM

【C. CSRF 防护测试】
C1. 验证 Django CSRF 中间件启用：
   - 不带 CSRF token 的 POST 请求应返回 403
   - 带正确 CSRF token 的 POST 请求正常处理
C2. API 接口 CSRF 豁免合理性：
   - 确认 /api/ 开头的接口使用 SessionAuthentication 或 TokenAuthentication
   - 确认非 API 接口（如表单提交）受 CSRF 保护

【D. 认证与授权测试】
D1. 越权访问：
   - 使用 session_token A 访问 session_token B 的测评结果 → 预期 403
   - 未付费用户访问 /api/reports/{id}/ → 预期 403
   - 普通用户访问 /api/admin/* → 预期 403
D2. Session 安全：
   - session_token 是否可预测（应为 UUID 或随机字符串）
   - 过期 session_token 访问 → 预期 401/403
D3. 支付安全：
   - 篡改订单金额 → 预期回调验证拒绝
   - 伪造支付回调签名 → 预期验证失败
   - 重复支付回调 → 预期幂等处理

【E. 数据加密测试】
E1. 传输加密：
   - 本地开发可使用 HTTP，生产环境必须 HTTPS
   - 验证 settings 中 SECURE_SSL_REDIRECT 配置
E2. 存储加密：
   - 测评原始答题数据是否加密存储（检查 Assessment 模型 answers 字段）
   - 用户设备指纹是否哈希存储
   - 支付订单签名是否使用 HMAC-SHA256
E3. 敏感数据脱敏：
   - 日志中不输出完整 session_token（应脱敏）
   - API 响应不返回内部数据库 ID（除必要的 assessment_id）
   - 错误信息不暴露堆栈跟踪（生产环境 DEBUG=False）

【F. 其他安全检查】
F1. HTTP 安全头：
   - X-Content-Type-Options: nosniff
   - X-Frame-Options: DENY 或 SAMEORIGIN
   - Content-Security-Policy 配置
F2. 依赖安全：
   - 运行 pip-audit 或 safety check 检查已知漏洞
F3. 文件上传安全：
   - 题库批量导入功能只接受 CSV 格式
   - 上传文件大小限制
```

### 5.2 隐私数据保护验证提示词

```
验证用户隐私数据（特别是心理测评数据）的保护措施：

1. 测评数据加密存储
   - 检查 Assessment 模型的 answers_raw 字段是否使用加密字段
   - 验证加密算法（AES-256 或 Django 的 Fernet 加密）
   - 验证解密仅在服务层进行，不通过 API 暴露

2. 数据最小化
   - API 响应只返回必要字段，不返回原始答题数据
   - 免费结果页不暴露完整 OCEAN 原始分（只返回百分位）
   - 分享链接不包含敏感测评数据

3. 数据保留策略
   - 验证是否有数据过期清理机制
   - 验证用户可请求删除自己的测评数据
   - 验证删除后数据不可恢复

4. 访问控制
   - 只有持有 session_token 的用户可访问自己的测评结果
   - 后台管理员可查看统计数据但不能查看个体原始答题
   - 日志中不记录具体答题内容

5. 第三方数据共享
   - 支付回调只传递订单号和金额，不传递测评内容
   - 分享功能只分享认证卡图片，不分享原始分数
```

### 5.3 验收标准

| 安全维度 | 标准 |
|---------|------|
| SQL 注入 | 零漏洞 |
| XSS | 零漏洞 |
| CSRF | 所有非 API POST 受保护 |
| 越权访问 | 零漏洞 |
| 数据加密 | 测评数据加密存储 |
| 支付安全 | 签名验证 + 金额校验 + 幂等 |
| 依赖漏洞 | 无高危漏洞 |
