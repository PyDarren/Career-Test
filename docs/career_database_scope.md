## 一、收录范围概述

### 1.1 数据来源

| 数据类别 | 来源 | 构建方式 | 更新频率 |
|---------|------|---------|---------|
| 职业基础信息 | 《中华人民共和国职业分类大典（2022 年版）》 | 人工筛选适合大学生/职场新人的职业方向，预计收录 80-120 个职业 | 每年一次 |
| MBTI 适配列表 | MBTI 官方职业适配指南 + 16Personalities 职业推荐 | 每种 MBTI 类型关联 10-15 个适配职业 | 初始构建后稳定 |
| 维度理想画像 | 职业心理学家评估 + 岗位 JD 分析 | 结合职业特性设定四维度的理想倾向分布 | 初始构建后稳定 |
| 认知功能适配 | 荣格认知功能理论 + 职业心理学研究 | 按主导认知功能归类适配职业 | 初始构建后稳定 |
| 薪资数据 | Boss 直聘、拉勾网公开薪资区间 | 抓取各职业的薪资中位数和分布范围，按城市分级标注 | 每季度更新 |
| 发展前景 | 行业报告 + 公开就业数据 | 人工撰写每个职业的 2-3 句发展前景描述 | 每半年更新 |

### 1.2 收录数量

- **预计收录**：80-120 个职业
- **筛选原则**：从《职业分类大典（2022 年版）》中筛选适合在校大学生和职场新人（18-28 岁）的职业方向，排除需长期专业资质准入的职业（如外科医生、飞行员等），聚焦应届可就业或 1-3 年内可达的职业方向

### 1.3 行业分类体系

数据库按 6 大行业类别组织，每类预计收录 13-20 个职业：

| 序号 | 行业类别 | 说明 | 预计职业数 |
|:---:|---------|------|:---------:|
| 1 | 商业/金融 | 商业运营、金融分析、市场营销、企业管理等 | 15-20 |
| 2 | 技术 | 软件开发、数据科学、产品管理、系统架构等 | 15-20 |
| 3 | 教育 | 教学、培训、课程开发、教育心理等 | 13-18 |
| 4 | 医疗保健 | 临床医疗、护理、心理咨询、公共卫生等 | 13-18 |
| 5 | 专业性职业 | 法律、人力资源、项目管理、精算等 | 13-18 |
| 6 | 创造性职业 | 设计、内容创作、建筑、影视音乐等 | 13-18 |
| | **合计** | | **80-120** |

## 二、数据库字段结构

每个职业记录包含以下 12 个字段：

| 序号 | 字段名 | 类型 | 说明 | 示例 |
|:---:|--------|------|------|------|
| 1 | `career_id` | string | 职业唯一标识 | `CAREER_001` |
| 2 | `career_name` | string | 职业名称 | `UI/UX 设计师` |
| 3 | `category` | string | 职业大类（6 大行业之一） | `创造性职业` |
| 4 | `mbti_fit` | JSON array | 适配 MBTI 类型列表 | `["ENFP","INFP","ENFJ","INFJ"]` |
| 5 | `mbti_ideal` | JSON object | 四维度理想倾向画像 | `{"E":60,"S":30,"T":40,"J":30}` |
| 6 | `cognitive_fit` | JSON array | 适配主导认知功能 | `["Ne","Ni","Fi"]` |
| 7 | `work_style` | string | 理想工作环境描述 | `鼓励创意、自主度高、团队协作` |
| 8 | `skill_required` | JSON array | 核心能力要求 | `["创意设计","用户研究","原型工具"]` |
| 9 | `salary_range` | string | 薪资参考范围（应届/1-3年） | `应届 8-12K，1-3年 12-20K` |
| 10 | `growth_prospect` | string | 发展前景描述 | `需求稳定，AI 工具推动岗位升级` |
| 11 | `description` | text | 职业简介（100-200 字） | `负责产品的交互设计和视觉呈现…` |
| 12 | `match_tags` | JSON array | 匹配关键词 | `["创意","审美","用户洞察"]` |

### 字段填写规范

- **`mbti_fit`**：列出 4-8 个最适配的 MBTI 类型，按适配度降序排列
- **`mbti_ideal`**：四维度倾向值（E/I、S/N、T/F、J/P），范围 0-100，50 为中立。例如 `{"E":60,"S":30,"T":40,"J":30}` 表示理想画像偏外向、直觉、情感、感知
- **`cognitive_fit`**：列出 2-4 个最适配的认知功能（如 Ne、Ni、Te、Fi 等）
- **`salary_range`**：格式统一为"应届 X-Y K，1-3年 A-B K"，数据来自 Boss 直聘/拉勾网上一季度公开数据
- **`description`**：100-200 字，涵盖工作内容、典型场景、入行路径
- **`match_tags`**：3-6 个关键词，用于结果页标签展示和匹配筛选

## 三、数据质量保障

### 3.1 初始构建阶段

- 由产品经理 + 心理学顾问协同构建初始数据库
- 完成后邀请 3-5 位高校就业指导老师进行交叉审核

### 3.2 上线前交叉验证

- **验证方法**：选取 30 个典型职业（见第四章），与已有平台（16Personalities、北森）的职业推荐结果进行对比
- **一致性标准**：MBTI 适配列表与对标平台的重合度 ≥ 70% 方可上线
- **验证流程**：对每个职业的 `mbti_fit` 列表，与 16Personalities 和北森对应职业的推荐类型逐一对比，计算重合率

### 3.3 持续优化

- 结果页设置"推荐不准"反馈按钮，用户可选择"方向完全不对"/"部分匹配"/"其他"
- 每月复盘反馈数据，修正量 > 5% 时触发数据库全面审查
- 薪资数据每季度自动更新，发展前景每半年更新

## 五、交叉验证执行方案

### 5.1 验证对象

- **验证职业数**：30 个（见第四章清单）
- **对标平台**：16Personalities（国际通用）、北森（国内权威）
- **验证维度**：每个职业的 `mbti_fit` 列表与对标平台推荐类型的重合度

### 5.2 验证流程

```
对每个职业：
  1. 查询 16Personalities 该职业的推荐 MBTI 类型列表 → 集合 A
  2. 查询北森该职业的推荐 MBTI 类型列表 → 集合 B
  3. 本数据库的 mbti_fit 列表 → 集合 C
  4. 计算一致性 = |C ∩ (A ∪ B)| / |C| × 100%
  5. 一致性 ≥ 70% → 通过
  6. 一致性 < 70% → 标记需修正，由心理学顾问复审

整体一致性 = 通过验证的职业数 / 30 × 100%
整体一致性 ≥ 70% → 数据库可上线
```

### 5.3 验证时间节点

- **执行时间**：阶段二任务 2.3（职业 fixture 数据编写完成后）
- **完成标志**：出具《职业数据库交叉验证报告》，整体一致性 ≥ 70%
