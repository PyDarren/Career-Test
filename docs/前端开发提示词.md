## 📐 第一步：16personalities.com 设计语言提取

以下参数通过分析 16personalities.com 提取，作为本项目视觉对标的基准。

### 1.1 全局设计基调

| 维度 | 16personalities 特征 | 本项目适配方案 |
|------|---------------------|---------------|
| **整体风格** | 现代、柔和、插画驱动、卡片式布局 | 保持，中文语境下增大字号和行距 |
| **圆角策略** | 大圆角为主（卡片 12-16px，按钮 8-12px，头像 100%） | 采用相同圆角体系 |
| **阴影策略** | 柔和漫射阴影（`0 2px 12px rgba(0,0,0,0.08)`），无硬边 | 保持相同阴影风格 |

| **交互风格** | 微动画温和不突兀，hover 有轻微提升效果 | 保持 |

## ⚠️ 严格约束（违反将导致代码被拒绝）

### 来自 AGENTS.md 的工程规范
1. **Django 模板架构**：使用 Django 模板引擎渲染，**禁止使用 Vue/React/Angular**。交互逻辑通过模板内嵌 `<script>` 或 `static/js/` 中的 vanilla JS 实现。
2. **模板渲染优先**：项目建设期为前后端不分离架构。
3. **计分算法不暴露**：前端代码中**禁止**实现或暴露测评计分算法，计分必须由后端完成。
4. **免登录架构**：测评提交、结果查看、免费卡片生成均不需要用户认证。使用 `device_fingerprint` + `session_token` 关联匿名用户。
5. **常量管理**：定价 2.99 元、题目数量 86 题等业务常量由后端通过 context 传入模板，前端不硬编码。
6. **静态资源路径**：所有图片/CSS/JS 使用 `{% static %}` 标签引用，禁止相对路径。
7. **吉祥物路径**：使用 `{% static 'images/mascots/' %}{{ type_code }}.png`，禁止使用设计稿原始路径。

### 来自 PRD.md 的功能约束
8. **免登录**：用户打开应用即可直接开始测评，页面无任何注册/登录入口。
9. **6 点量表**：答题界面使用 6 点强制选择格式，配渐变色选项（无中性中间点）。
